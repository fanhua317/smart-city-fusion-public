# 实验注意事项

## 当前结论

当前官方可上传的最稳版本是 `33_mix_avg21_45`。它已经由 `Evaluation.exe` 成功生成 `.dt`，并被固化为本地 final 版本：

- 官方评测文件：`results/final/eval/rank_best_33_mix_avg21_45.dt`

- 当前分数表：`results/score_tables/current_official_scores.csv`

## 不要提交到 GitHub 的内容

以下内容只保留在本机，不建议上传到公开仓库：

- `target/`：测试集图像，属于课程数据。
- `Evaluation.exe`：官方评测工具，体积较大且来源属于课程环境。
- `results/candidates*`、`results/top_candidates`、`results/final/fused`：批量生成图像，体积很大。
- `*.dt`：官方评测工具生成的加密提交文件，不手工编辑，不公开传播。
- `leaderboard_*.csv`：包含排行榜用户和学号信息，除非仓库明确为私有且确有必要，否则不要上传。

## 生成与评测注意

1. 输出必须与 `target/ir`、`target/vi` 同名、同数量、同尺寸。
2. 最终目录必须只有 93 张可解码图片，不要混入 `params.json`、`validation.json` 等元数据文件。
3. `Evaluation.exe` 输出文件名不要手动加 `.dt`，工具会自动追加扩展名；例如填写 `rank_best_33_mix_avg21_45`。
4. 第三轮部分候选本地指标很好，但曾在官方工具中触发内存分配错误，不能只看本地 CSV 决定上传。
5. `.dt` 是最终上传文件，不要尝试解密、伪造或手工修改。

## 当前最佳指标画像

`33_mix_avg21_45` 的优势是结构保真：`MSE`、`PSNR`、`SSIM`、`Nabf`、`CC`、`SCD` 比较稳。短板主要是 `AG`、`EN`、`MI`、`SD`、`SF`、`Qabf`。

后续优化应先保证能通过 `Evaluation.exe` 生成 `.dt`，再谈冲榜。低风险方向是以 `33_mix_avg21_45` 为结构锚，逐步增强细节和信息量，而不是直接用本地指标很激进的候选。
